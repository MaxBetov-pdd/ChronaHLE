# Security policy

Please do not disclose a security vulnerability in a public issue. Use
[GitHub private vulnerability reporting](https://github.com/MaxBetov-pdd/ChronaHLE/security/advisories/new)
and include the affected ChronaHLE version, host platform and reproduction
details.

Only the latest released version and the current `main` branch receive security
fixes. Compatibility bugs that do not create a host security risk belong in the
normal issue tracker.
